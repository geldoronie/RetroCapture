# RetroCapture 0.6.0-alpha

Eleventh alpha release. Big reliability pass on the recording/streaming pipeline,
a major web portal overhaul, named profiles for both streaming and recording,
and final bits of RetroArch GLSL spec coverage.

**Compared to 0.5.0-alpha**: 11 PRs ┬╖ 76 files changed ┬╖ +9,732 / тИТ2,496 lines.

---

## Highlights

### Recording & streaming reliability

- **Recording desync fixed at higher quality presets** (#38). Root cause was
  PTS truncation from `time_base = {1, fps}`; bumped to `{1, 90000}` so PTS
  granularity matches the encoder's expectations. `medium + 12 Mbps` and above
  now stay in sync end-to-end.
- **Streaming hardening** (#41): independent audio drain (no longer gated
  by sync zone), non-blocking TCP send (`MSG_DONTWAIT`), simpler client
  back-pressure handling. No more audio dropouts during long sessions.
- **Render pipeline latency** (#27): tightened `glFinish` + sync `glReadPixels`
  polling to reduce frame-to-encode latency.
- **Recording A/V desync + shutdown stability** (#28): backpressure across the
  encoding stage, clean shutdown when stopping mid-encode.

### Named profiles for streaming & recording

- **Recording profiles** (#39) тАФ save / load / delete / apply complete
  recording settings (codec, bitrate, FPS, shader-apply flag, audioтАж) by name.
  Stored as JSON under `~/.local/share/retrocapture/recording_profiles/`
  (Linux) / `%APPDATA%\RetroCapture\data\recording_profiles\` (Windows).
- **Streaming profiles** (#42) тАФ same UX, parallel storage under
  `streaming_profiles/`.
- Both surfaces (native UI + web portal) can manage profiles.

### Web portal overhaul (#43)

- **Three dedicated pages** with shared top navigation: **Home** (live stream
  player + status), **Recordings** (browse / delete / download), **Configuration**
  (everything else).
- **Live MPEG-TS player** on Home, vendored `mpegts.js` + native fallback.
- **Bootstrap + Bootstrap Icons fully vendored** тАФ portal works fully offline
  via service worker (`v3`).
- **Master shader pipeline toggle** тАФ bypass the active preset without losing it.
- **Per-pipeline shader override** тАФ stream and recording can independently opt
  out of the shader, taking the source frame instead of the processed one.
- **Editable preset shader parameters** via modal (was native-UI-only).
- **Source-tab parity**: V4L2 retro-console preset row matches native UI;
  duplicated hardcoded V4L2 controls block was removed.
- **Overscan controls** exposed in the web UI.
- Portal active-state stays coherent when streaming hosts the portal.

### Shader spec coverage (#29)

- Closed remaining RetroArch GLSL spec gaps: **PassFeedback** and the
  **OriginalHistory** alias chain. More upstream presets now load correctly.

### Capture presets persist parameter overrides (#33)

- Capture presets now actually save the shader parameter overrides you tweak
  after applying a preset (previously the overrides were re-loaded from the
  preset file every time, silently discarding edits).

### Storage paths standardized (#32)

- Linux follows **XDG Base Directory** (config / data / cache / videos).
- Windows follows **Known Folders** (`SHGetFolderPath` for Roaming AppData,
  Local AppData, My Videos).
- Per-getter env overrides (`RETROCAPTURE_CONFIG_DIR`, `_DATA_DIR`, `_CACHE_DIR`,
  `_ASSETS_DIR`) for AppImage / CI / packagers.
- One-shot idempotent migration of legacy `~/.config/retrocapture/assets` тЖТ
  `~/.local/share/retrocapture/`.

### CPU compatibility mode for x86-64 (#20)

- Default Docker builds now target a portable baseline (no AVX2). AVX-less
  CPUs (older Sandy/Ivy Bridge, low-power x86, virt CPUs) work out of the box.
- `--native` flag remains for local builds that want `-march=native`.

## Upgrade notes

- Existing config in `~/.config/retrocapture/config.json` keeps working.
- First run of 0.6.0 migrates legacy `assets/` and `ssl/` from `~/.config` to
  `~/.local/share/retrocapture/` (Linux) / `%APPDATA%\RetroCapture\data\`
  (Windows). Idempotent тАФ marker file `MIGRATED.txt` prevents re-runs.
- Existing capture presets keep loading. Re-save them once if you want the
  current shader-parameter overrides to stick (was the bug fixed in #33).
